# AI中台 · 客户接入指南

> 本文档面向：客户方开发团队
>
> 目标：帮助客户方完成与 AI中台服务的对接，包括需要提供的接口、需要调用的接口、前端集成方式
>

---

## 一、整体对接流程

```
企业客户方 APS 前端
      │
      │ ① 调用 AI 网关触发插单评估
      │ ② 轮询评估状态
      │ ③ 收到 AI 评估结果展示给用户
      ▼
  AI 网关服务（端口 8000）
      │
      │ ① 调用客户 APS 接口获取数据（插单列表、受影响订单）
      │ ② 调用 AI服务工作流进行分析
      ▼
  企业客户方 APS 后台接口（提供给 AI 网关调用）
```

**你们需要做两件事：**

| 事项 | 说明 |
|------|------|
| **A. 提供接口** | 实现 APS 数据查询接口，供 AI 网关调用（参见第二节接口清单） |
| **B. 调用接口** | 在前端调用 AI 网关接口，获取 AI 分析结果（参见第三节接口说明） |

> **注意**：AI 网关需要调用 APS 的接口获取数据，请确保这些接口可被 AI 网关服务器访问。

## 二、需要提供的接口（A）

> AI 中台服务需要读取 APS 系统的数据，请按以下规范实现并提供。

### 2.1 统一规范

```
Base URL：例如 http://your-aps-host/api
请求格式：application/json
响应格式：application/json
鉴权方式：Bearer Token（Token 由双方协商）
```

**统一响应结构：**

```json
{
  "code": 0,
  "message": "ok",
  "data": { }
}
```

### B1. 触发插单评估（启动评估任务）

**接口**：`POST /ai/rush-order/evaluate/start`（我方网关地址）

**说明**：前端调用此接口，触发 AI 助手对当前插单数据进行评估分析。

**请求体：** 无（通过 Header Authorization 传递登录 token）

**响应示例：**

```json
  "code": 0,
```

| 字段 | 类型 | 说明 |
|------|------|------|
| taskId | String | 评估任务 ID，用于后续查询状态和获取结果 |
| status | String | 任务状态，固定返回 `running` |

---

### B2. 轮询评估状态

**接口**：`GET /ai/rush-order/evaluate/status/{taskId}`（我方网关地址）

**说明**：前端每隔 3 秒轮询此接口，直到状态变为 `completed`。

**响应示例（运算中）：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "taskId": "eval_abc12345",
    "status": "running",
    "message": "正在评估中，请稍候..."
  }
}
```

**响应示例（完成）：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "taskId": "eval_abc12345",
    "status": "completed"
  }
}
```

| status 值 | 说明 |
|-----------|------|
| running | 运算中，继续轮询 |
| completed | 运算完成，可调用 B3 获取 AI 报告 |

---

### B3. 获取 AI 分析报告（流式）

**接口**：`POST /ai/rush-order/evaluate/analyze/{taskId}/stream`（我方网关地址）

**说明**：评估完成后，调用此接口获取 AI 分析报告（流式 SSE 响应）。

**流式响应格式：**

```
data: {"type":"chunk","content":"第一段文字"}
data: {"type":"chunk","content":"第二段文字"}
...
data: {"type":"done","conversationId":"conv_abc123","taskId":"eval_abc12345"}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| type | String | `chunk`=文字块，`done`=完成，`error`=错误 |
| content | String | AI 返回的文字内容（type=chunk 时） |
| conversationId | String | 对话 ID，用于后续追问（type=done 时） |
| taskId | String | 任务 ID（type=done 时） |

### 4.3 手动调用 API（不使用 SDK）

```javascript
const AI_SERVICE_URL = 'http://ai-service-host:8000'  // 替换为我方网关地址

/**
 * 完整插单评估流程（手动调用）
 */
async function runRushOrderEvaluation(token, onStatusChange, onChunk, onDone, onError) {
  try {
    // 第一步：触发评估
    onStatusChange('正在提交评估请求...')
    const startRes = await fetch(`${AI_SERVICE_URL}/ai/rush-order/evaluate/start`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}`
      }
    })
    const startData = await startRes.json()
    if (startData.code !== 0) throw new Error(startData.message)
    const taskId = startData.data.taskId

    // 第二步：轮询状态
    onStatusChange('APS 正在运算，请稍候...')
    let completed = false
    for (let i = 0; i < 100 && !completed; i++) {
      await sleep(3000)
      const statusRes = await fetch(`${AI_SERVICE_URL}/ai/rush-order/evaluate/status/${taskId}`, {
        headers: { 'Authorization': `Bearer ${token}` }
      })
      const statusData = await statusRes.json()
      if (statusData.data.status === 'completed') {
        completed = true
      } else {
        onStatusChange(statusData.data.message || '运算中...')
      }
    }
    if (!completed) throw new Error('运算超时')

    // 第三步：流式获取 AI 报告
    onStatusChange('正在生成 AI 分析报告...')
    const resp = await fetch(`${AI_SERVICE_URL}/ai/rush-order/evaluate/analyze/${taskId}/stream`, {
      method: 'POST',
      headers: { 'Authorization': `Bearer ${token}` }
    })

    const reader = resp.body.getReader()
    const decoder = new TextDecoder('utf-8')
    let buffer = '', fullAnswer = '', conversationId = ''

    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      buffer += decoder.decode(value, { stream: true })
      const lines = buffer.split('\n')
      buffer = lines.pop()
      for (const line of lines) {
        if (!line.startsWith('data: ')) continue
        try {
          const event = JSON.parse(line.slice(6))
          if (event.type === 'chunk') {
            fullAnswer += event.content
            onChunk && onChunk(event.content)
          } else if (event.type === 'done') {
            conversationId = event.conversationId || ''
            onDone && onDone(fullAnswer, conversationId)
          } else if (event.type === 'error') {
            onError && onError(event.message)
          }
        } catch (e) { continue }
      }
    }
  } catch (err) {
    onError && onError(err.message)
  }
}

function sleep(ms) { return new Promise(r => setTimeout(r, ms)) }
```

### 4.4 报表页面

评估完成后，用户可查看详细的 HTML 报表，包含：
- **总览统计**：插单数量、受影响订单数、造成延迟数、正常数
- **插单明细表**：编号、人员、物料、客户、数量、优先级、评估结果
- **受影响订单表**：订单号、物料、交期、原排产、新排产、影响天数

**报表访问地址**：`GET /api/report`

```
# 直接访问（浏览器打开）
http://ai-gateway:8000/api/report

# 带 Token 访问
http://ai-gateway:8000/api/report?token=mock_token_valid
```

> **注意**：报表数据来自评估结果，查看报表前需先触发一次插单评估。

### 4.5 SDK 配置参数

```javascript
window.APS_CONFIG = {
  BASE_URL: 'http://ai-gateway:8000',        // 必填：统一服务地址
  REPORT_BASE_URL: 'http://report-server:8080'  // 可选：独立的报表服务地址
}
```

| 配置项 | 必填 | 说明 |
|--------|------|------|
| `BASE_URL` | 是 | 统一服务地址（端口 8000） |
| `REPORT_BASE_URL` | 否 | 报表服务地址，未配置时从 BASE_URL 推导 |

**报表地址生成规则**：
1. 优先使用 `REPORT_BASE_URL`：`${REPORT_BASE_URL}/api/report`
2. 未配置时从 `BASE_URL` 推导：`${BASE_URL}/api/report`（将 `/ai` 替换为 `/api`）

```
      特别检查：remark 字段不为 null，productScheduleEndDate 和 lackDay 有值

□ 5. POST /api/aps/affect-order/paging
      确认返回受影响订单列表，delayDay 字段正确

□ 6. GET /api/aps/order?orderNo=xxx
      传入真实订单号，确认返回数据

□ 7. GET /api/aps/shortage
      确认返回缺料列表，字段完整

□ 8. GET /api/aps/capacity
      确认返回资源负荷数据，loadRate 和 status 逻辑正确

□ 9. GET /api/aps/risk
      确认返回风险订单，riskLevel 判断逻辑正确

□ 10. 前端联调（使用 SDK）：触发评估 → 轮询等待 → 收到 AI 回答
      确认全流程无异常，AI 回答内容符合预期

□ 11. GET /api/report
      触发评估后访问报表页面，确认显示插单明细和受影响订单数据
```

## 六、常见对接问题

**Q1：接口返回的 null 字段导致 AI 回答异常**

部分字段（`remark`、`materialBottlenecks`、`capacityBottlenecks`）必须按规范返回空字符串或空数组，不能返回 `null`。

```json
// ❌ 错误
"remark": null,
"materialBottlenecks": null

// ✅ 正确
"remark": "",
"materialBottlenecks": []
```

**Q2：gapDays 正负值定义**

```
gapDays = scheduledDate - dueDate（天数差）
正数 = 模拟交期比期望交期晚 = 延迟
负数 = 模拟交期比期望交期早 = 提前
```

**Q3：delayedTop10 排序要求**

`delayedTop10` 数组必须按 `delayDays` 降序排列（延迟天数最多的排在最前）。

**Q4：前端 CORS 问题**

如果前端直接调用 AI 助手服务出现跨域，建议通过你们后端做代理转发，避免将 AI 服务地址暴露在前端。

**Q5：APS 运算失败如何处理**

当 `/rush-order/simulate/{id}/status` 返回 `status: failed` 时，`data` 中包含 `failReason` 字段描述失败原因。请展示给用户并提示重试。

**Q6：报表页面 URL 是什么？**

报表页面地址是 `/api/report`，不是 `/report`。SDK 中的 `getReportUrl()` 会自动生成正确的地址。

如果报表服务与统一服务不在同一地址，可通过 `REPORT_BASE_URL` 配置：
```javascript
window.APS_CONFIG = {
  BASE_URL: 'http://ai-gateway:8000',
  REPORT_BASE_URL: 'http://report-server:8080'  // 独立报表服务地址
}
```

**Q7：报表没有数据怎么办？**

报表数据来自 `rush-order/paging` 和 `affect-order/paging` 接口。需要先触发一次插单评估（调用 `/api/aps/rush-order-evaluate`），等评估完成后再访问报表。
