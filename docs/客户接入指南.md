# APS AI 助手 · 客户接入指南

> 本文档面向：客户方开发团队
>
> 目标：帮助客户方完成与 AI 助手服务的对接，包括需要提供的接口、需要调用的接口、前端集成方式
>
> 阅读时间：约 20 分钟

---

## 一、整体对接流程

```
企业客户方 APS 前端（你们的前端）
      │
      │ ① 用户填写插单 → 调用 AI 助手服务触发分析
      │ ② 轮询等待运算完成
      │ ③ 收到 AI 评估结果展示给用户
      ▼
  AI 助手服务（我方提供）
      │
      │ ① 调用客户 APS 接口获取数据
      │ ② 调用 Dify AI 分析
      ▼
  企业客户方 APS 后台接口（你们需要实现）
```

**你们需要做两件事：**

| 事项 | 说明 |
|------|------|
| **A. 提供接口** | 实现 APS 数据查询接口，供 AI 助手服务调用 |
| **B. 调用接口** | 在前端/后端调用 AI 助手服务接口，获取 AI 分析结果 |

## 二、需要你们提供的接口（A）

> AI 助手服务需要读取你们 APS 系统的数据，请按以下规范实现并提供。

### 2.1 统一规范

```
Base URL：由你们提供，例如 http://your-aps-host/api
请求格式：application/json
响应格式：application/json
鉴权方式：Bearer Token（Token 由双方协商）
```

**统一响应结构：**

```json
{
  "code": 0,
  "message": "ok",
  "data": { }
}
```

### 2.2 接口列表

#### A1. 触发插单运算

**接口**：`POST /rush-order/simulate`

**说明**：用户填写插单信息后，调用此接口触发 APS 后台运算。

**请求体：**

```json
{
  "rushOrders": [
    {
      "customerCode": "C001",
      "materialCode": "M20240001",
      "quantity": 100,
      "dueDate": "2025-09-01",
      "priority": "High",
      "remark": "加急处理"
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| rushOrders | Array | 是 | 插单列表，至少 1 条 |
| customerCode | String | 是 | 客户编码 |
| materialCode | String | 是 | 物料编码 |
| quantity | Number | 是 | 需求数量 |
| dueDate | String | 是 | 期望交期，格式 `yyyy-MM-dd` |
| priority | String | 否 | 优先级，默认 `Normal`，可选 `High` / `Normal` / `Low` |
| remark | String | 否 | 备注 |

**响应示例：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "simulationId": "SIM20260329001",
    "status": "queued",
    "queuePosition": 1,
    "estimatedSeconds": 30
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| simulationId | String | 本次运算任务唯一 ID，后续所有接口都需要用到 |
| status | String | 初始固定返回 `queued` |
| queuePosition | Number | 排队位置，1 表示立即执行 |
| estimatedSeconds | Number | 预估等待秒数 |

---

#### A2. 查询运算状态

**接口**：`GET /rush-order/simulate/{simulationId}/status`

**说明**：触发运算后，前端每隔 3 秒轮询此接口，直到状态变为 `completed`。

**响应示例（运算中）：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "simulationId": "SIM20260329001",
    "status": "running",
    "progress": 60
  }
}
```

**响应示例（完成）：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "simulationId": "SIM20260329001",
    "status": "completed",
    "progress": 100
  }
}
```

| status 值 | 说明 |
|-----------|------|
| queued | 排队等待 |
| running | 运算中 |
| completed | 运算完成，可查询结果 |
| failed | 运算失败，见 failReason 字段 |

## 四、前端集成参考代码

### 4.1 完整插单评估流程

```javascript
const AI_SERVICE_URL = 'http://ai-service-host:8001/api'  // 替换为我方网关地址
const APS_SERVICE_URL = 'http://your-aps-host/api'        // 替换为你们APS服务地址

/**
 * 插单评估完整流程
 */
async function runRushOrderEvaluation(rushOrders, userId, onStatusChange, onResult, onError) {
  try {
    // 第一步：触发 APS 运算
    onStatusChange('正在提交插单，请稍候...')
    const triggerRes = await fetch(`${APS_SERVICE_URL}/rush-order/simulate`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ rushOrders })
    })
    const triggerData = await triggerRes.json()
    if (triggerData.code !== 0) {
      onError('提交失败：' + triggerData.message); return
    }

    const simulationId = triggerData.data.simulationId
    onStatusChange('APS 正在运算，请稍候...')

    // 第二步：轮询运算状态（每3秒，最多5分钟）
    const completed = await pollStatus(simulationId, onStatusChange)
    if (!completed) { onError('运算超时，请稍后重试'); return }

    // 第三步：调用 AI 助手生成分析报告
    onStatusChange('正在生成 AI 分析报告...')
    const aiRes = await fetch(`${AI_SERVICE_URL}/ai/conversation`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        scene: 'rush_order_eval',
        simulationId,
        userId,
        conversationId: '',
        message: ''
      })
    })
    const aiData = await aiRes.json()
    if (aiData.code !== 0) {
      onError('AI 分析失败：' + aiData.message); return
    }

    window._currentConversationId = aiData.data.conversationId
    window._currentSimulationId = simulationId
    onResult(aiData.data.answer, aiData.data.reportUrl)

  } catch (err) {
    onError('网络错误：' + err.message)
  }
}

/**
 * 轮询运算状态
 */
async function pollStatus(simulationId, onStatusChange) {
  const maxRetries = 100
  for (let i = 0; i < maxRetries; i++) {
    await sleep(3000)
    const res = await fetch(`${APS_SERVICE_URL}/rush-order/simulate/${simulationId}/status`)
    const data = await res.json()
    const status = data.data.status

    if (status === 'completed') return true
    if (status === 'failed') {
      throw new Error('APS 运算失败：' + (data.data.failReason || '未知原因'))
    }
    onStatusChange(`APS 运算中，请稍候... (${i + 1})`)
  }
  return false
}

/**
 * 用户追问
 */
async function followUp(message, userId, onResult) {
  const res = await fetch(`${AI_SERVICE_URL}/ai/conversation`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      scene: 'rush_order_eval',
      simulationId: window._currentSimulationId,
      userId,
      conversationId: window._currentConversationId,
      message
    })
  })
  const data = await res.json()
  onResult(data.data.answer, data.data.reportUrl)
}

function sleep(ms) { return new Promise(r => setTimeout(r, ms)) }
```

## 五、接口联调验证清单

对接完成后，请按以下顺序逐项验证：

```
□ 1. POST /rush-order/simulate
      传入 1 条插单，确认返回 simulationId，无报错

□ 2. GET /rush-order/simulate/{id}/status
      轮询 3~5 次，确认状态从 queued → running → completed

□ 3. GET /rush-order/simulate/{id}/result
      确认返回结构完整，rushOrders 字段符合规范
      特别检查：remark 不为 null，bottlenecks 不为 null

□ 4. GET /aps/order?orderNo=xxx
      传入真实订单号，确认返回数据

□ 5. GET /aps/shortage
      确认返回缺料列表，字段完整

□ 6. GET /aps/capacity
      确认返回资源负荷数据，loadRate 和 status 逻辑正确

□ 7. GET /aps/risk
      确认返回风险订单，riskLevel 判断逻辑正确

□ 8. 前端联调：填写插单 → 轮询等待 → 收到 AI 回答
      确认全流程无异常，AI 回答内容符合预期
```

## 六、常见对接问题

**Q1：接口返回的 null 字段导致 AI 回答异常**

部分字段（`remark`、`materialBottlenecks`、`capacityBottlenecks`）必须按规范返回空字符串或空数组，不能返回 `null`。

```json
// ❌ 错误
"remark": null,
"materialBottlenecks": null

// ✅ 正确
"remark": "",
"materialBottlenecks": []
```

**Q2：gapDays 正负值定义**

```
gapDays = scheduledDate - dueDate（天数差）
正数 = 模拟交期比期望交期晚 = 延迟
负数 = 模拟交期比期望交期早 = 提前
```

**Q3：delayedTop10 排序要求**

`delayedTop10` 数组必须按 `delayDays` 降序排列（延迟天数最多的排在最前）。

**Q4：前端 CORS 问题**

如果前端直接调用 AI 助手服务出现跨域，建议通过你们后端做代理转发，避免将 AI 服务地址暴露在前端。

**Q5：APS 运算失败如何处理**

当 `/rush-order/simulate/{id}/status` 返回 `status: failed` 时，`data` 中包含 `failReason` 字段描述失败原因。请展示给用户并提示重试。

## 附录：错误码说明

| HTTP 状态码 | code | 说明 | 处理建议 |
|-------------|------|------|----------|
| 200 | 0 | 成功 | — |
| 400 | INVALID_PARAM | 参数缺失或格式错误 | 检查请求字段 |
| 404 | SIMULATION_NOT_FOUND | simulationId 不存在 | 检查 ID 是否正确 |
| 409 | QUEUE_CONFLICT | 运算队列已满 | 稍后重试 |
| 422 | APS_CALC_FAILED | APS 运算失败 | 检查输入数据合法性 |
| 500 | INTERNAL_ERROR | 服务内部错误 | 联系 AI 服务方排查 |
