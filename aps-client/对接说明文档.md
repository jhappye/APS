# AI中台 · 对接说明文档

> **文档面向**：客户方开发团队
> **目标**：帮助客户方完成与 AI中台服务的对接，包括需要提供的接口、需要调用的接口、前端集成方式  
> **阅读时间**：约 20 分钟

---

## 一、整体对接流程

```
企业客户方 APS 前端
      │
      │ ① 用户填写插单 → 调用 AI 助手服务触发分析
      │ ② 轮询等待运算完成
      │ ③ 收到 AI 评估结果展示给用户
      ▼
  AI 助手服务
      │
      │ 需要调用企业客户方暴露的接口获取 APS 数据
      ▼
  企业客户方 APS 后台接口（客户方需要实现）
```

**企业客户方需要做两件事：**

| 事项          | 说明                            |
| ----------- | ----------------------------- |
| **A. 暴露接口** | 提供 APS 数据查询接口，供 AI 助手服务调用     |
| **B. 调用接口** | 在前端/后端调用 AI 助手服务接口，获取 AI 分析结果 |

---

## 二、企业客户方需要暴露的接口（A）

> AI 助手服务需要读取企业客户方的 APS 数据，请按以下规范实现并提供。

### 统一规范

```
Base URL：由客户方提供，例如 http://your-aps-host/api
请求格式：application/json
响应格式：application/json
鉴权方式：Bearer Token（Token 由双方协商）
```

**统一响应结构：**

```json
{
  "code": 0,
  "message": "ok",
  "data": { }
}
```

### A1. 触发插单运算

**接口**：`POST /rush-order/simulate`

**说明**：用户填写插单信息后，调用此接口触发 APS 后台运算。

**请求体：**

```json
{
  "rushOrders": [
    {
      "customerCode": "C001",
      "materialCode": "M20240001",
      "quantity": 100,
      "dueDate": "2025-09-01",
      "priority": "High",
      "remark": "加急处理"
    }
  ]
}
```

**请求字段说明：**

| 字段           | 类型     | 必填  | 说明                                           |
| ------------ | ------ | --- | -------------------------------------------- |
| rushOrders   | Array  | 是   | 插单列表，至少 1 条                                  |
| customerCode | String | 是   | 客户编码                                         |
| materialCode | String | 是   | 物料编码                                         |
| quantity     | Number | 是   | 需求数量                                         |
| dueDate      | String | 是   | 期望交期，格式 `yyyy-MM-dd`                         |
| priority     | String | 否   | 优先级，默认 `Normal`，可选 `High` / `Normal` / `Low` |
| remark       | String | 否   | 备注                                           |

**响应示例：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "simulationId": "SIM20260329001",
    "status": "queued",
    "queuePosition": 1,
    "estimatedSeconds": 30
  }
}
```

**响应字段说明：**

| 字段               | 类型     | 说明                      |
| ---------------- | ------ | ----------------------- |
| simulationId     | String | 本次运算任务唯一 ID，后续所有接口都需要用到 |
| status           | String | 初始固定返回 `queued`         |
| queuePosition    | Number | 排队位置，1 表示立即执行           |
| estimatedSeconds | Number | 预估等待秒数                  |

### A2. 查询运算状态

**接口**：`GET /rush-order/simulate/{simulationId}/status`

**说明**：触发运算后，前端每隔 3 秒轮询此接口，直到状态变为 `completed`。

**路径参数：**

| 参数           | 说明              |
| ------------ | --------------- |
| simulationId | 由 A1 接口返回的任务 ID |

**响应示例（运算中）：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "simulationId": "SIM20260329001",
    "status": "running",
    "progress": 60
  }
}
```

**响应示例（完成）：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "simulationId": "SIM20260329001",
    "status": "completed",
    "progress": 100
  }
}
```

**status 取值：**

| 值         | 说明                   |
| --------- | -------------------- |
| queued    | 排队等待                 |
| running   | 运算中                  |
| completed | 运算完成，可查询结果           |
| failed    | 运算失败，见 failReason 字段 |

| 字段        | 类型     | 说明                       |

## 三、客户方需要调用的接口（B）

> 以下接口由AI方提供，客户方前端/后端调用。

### B1. 发起 AI 插单评估对话

**接口**：`POST /ai/rush-order/evaluate/analyze/{taskId}/stream`（我方地址）

**说明**：触发评估后，前端调用此接口获取 AI 分析报告（流式）。该接口会调用 AI服务工作流，传入插单数据和受影响订单数据，返回 AI 分析结果。

**路径参数：**

| 参数    | 说明                   |
| ----- | -------------------- |
| taskId | 由 `POST /ai/rush-order/evaluate/start` 接口返回的任务 ID |

**请求体：** 无需请求体，通过 Header 传递 Authorization

**流式响应格式：**

```
data: {"type":"chunk","content":"第一段文字"}
data: {"type":"chunk","content":"第二段文字"}
...
data: {"type":"done","conversationId":"conv_abc123","taskId":"eval_xxx"}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| type | String | `chunk`=文字块，`done`=完成，`error`=错误 |
| content | String | AI 返回的文字内容（type=chunk 时） |
| conversationId | String | 对话 ID，用于后续追问（type=done 时） |
| taskId | String | 任务 ID（type=done 时） |
| message | String | 错误信息（type=error 时） |

**响应示例（完成时）：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "answer": "根据您输入的 **2** 个插单需求，已完成模拟分析...",
    "conversationId": "conv_abc123",
    "taskId": "eval_xxx"
  }
}
```

**注意**：该接口为流式 SSE 响应，上表为最终完成时返回的数据结构。

## 四、前端集成参考代码

> 以下代码为参考示例，使用 JavaScript（适用于 Vue / React 等主流框架）。

### 完整插单评估流程

```javascript
const AI_SERVICE_URL = 'http://ai-service-host/api/v1'  // 替换为我方服务地址

/**
 * 插单评估完整流程
 * @param {string} taskId - 评估任务 ID（由 POST /ai/rush-order/evaluate/start 返回）
 * @param {function} onStatusChange - 状态回调：(status: string) => void
 * @param {function} onChunk - 流式文字块回调：(chunk: string) => void
 * @param {function} onDone - 完成回调：(fullAnswer: string, conversationId: string) => void
 * @param {function} onError - 错误回调：(error: string) => void
 */
async function runRushOrderEvaluation(taskId, onStatusChange, onChunk, onDone, onError) {
  try {
    // 直接调用流式 AI 报告接口
    onStatusChange('正在生成 AI 分析报告...')
    const resp = await fetch(`${AI_SERVICE_URL}/ai/rush-order/evaluate/analyze/${taskId}/stream`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${window._gatewayToken}`  // 登录时获取的 token
      }
    })

    if (!resp.ok) {
      throw new Error(`请求失败 ${resp.status}`)
    }

    const reader = resp.body.getReader()
    const decoder = new TextDecoder('utf-8')
    let buffer = '', fullAnswer = '', conversationId = ''

    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      buffer += decoder.decode(value, { stream: true })
      const lines = buffer.split('\n')
      buffer = lines.pop()

      for (const line of lines) {
        if (!line.startsWith('data: ')) continue
        try {
          const event = JSON.parse(line.slice(6))
          if (event.type === 'chunk') {
            fullAnswer += event.content
            onChunk && onChunk(event.content)
          } else if (event.type === 'done') {
            conversationId = event.conversationId || ''
            onDone && onDone(fullAnswer, conversationId)
          } else if (event.type === 'error') {
            onError && onError(event.message)
          }
        } catch (e) {
          continue
        }
      }
    }
  } catch (err) {
    onError && onError(err.message)
  }
}
```

**注意**：上述代码适用于使用 SDK 的场景。如使用 SDK（`aps-ai-sdk.js`），只需调用 `ApsAI.evaluateRushOrders(callbacks)` 即可完成完整流程。

### Vue 3 组件示例

```vue
<template>
  <div class="rush-order-panel">
    <!-- 状态提示 -->
    <div v-if="statusMsg" class="status-tip">
      <span class="loading-icon">⏳</span> {{ statusMsg }}
    </div>

    <!-- AI 回答展示 -->
    <div v-if="aiAnswer" class="ai-answer">
      <div v-html="renderMarkdown(aiAnswer)"></div>

      <!-- 追问输入框 -->
      <div class="followup-bar">
        <input
          v-model="followUpMsg"
          placeholder="回复"是"查看详细数据，或输入其他问题..."
          @keyup.enter="sendFollowUp"
        />
        <button @click="sendFollowUp">发送</button>
      </div>

      <!-- 报表链接 -->
      <a v-if="reportUrl" :href="reportUrl" target="_blank" class="report-link">
        📊 查看详细数据报表
      </a>
    </div>
  </div>
</template>

<script setup>
import { ref } from 'vue'

const statusMsg = ref('')
const aiAnswer = ref('')
const reportUrl = ref('')
const followUpMsg = ref('')

// 启动插单评估（由父组件调用，传入 rushOrders 和 userId）
async function startEvaluation(rushOrders, userId) {
  aiAnswer.value = ''
  reportUrl.value = ''

  await runRushOrderEvaluation(
    rushOrders,
    userId,
    (status) => { statusMsg.value = status },
    (answer, url) => {
      statusMsg.value = ''
      aiAnswer.value = answer
      reportUrl.value = url
    },
    (error) => {
      statusMsg.value = ''
      aiAnswer.value = `❌ ${error}`
    }
  )
}

// 用户追问
async function sendFollowUp() {
  if (!followUpMsg.value.trim()) return
  const msg = followUpMsg.value
  followUpMsg.value = ''

  await followUp(msg, 'current_user_id', (answer, url) => {
    aiAnswer.value = answer
    if (url) reportUrl.value = url
  })
}

// 简单 Markdown 渲染（加粗、换行）
function renderMarkdown(text) {
  return text
    .replace(/\*\*(.*?)\*\*/g, '<strong>$1</strong>')
    .replace(/\n/g, '<br/>')
}

defineExpose({ startEvaluation })
</script>
```

## 五、接口联调验证清单

对接完成后，请按以下顺序逐项验证：

```
□ 1. POST /rush-order/simulate
      传入 1 条插单，确认返回 simulationId，无报错

□ 2. GET /rush-order/simulate/{id}/status
      轮询 3~5 次，确认状态从 queued → running → completed

□ 3. GET /rush-order/simulate/{id}/result
      确认返回结构完整，rushOrders 字段符合规范
      特别检查：remark 不为 null，bottlenecks 不为 null

□ 4. GET /aps/order?orderNo=xxx
      传入真实订单号，确认返回数据

□ 5. GET /aps/shortage
      确认返回缺料列表，字段完整

□ 6. GET /aps/capacity
      确认返回资源负荷数据，loadRate 和 status 逻辑正确

□ 7. GET /aps/risk
      确认返回风险订单，riskLevel 判断逻辑正确

□ 8. 前端联调：填写插单 → 轮询等待 → 收到 AI 回答
      确认全流程无异常，AI 回答内容符合预期
```

## 六、常见对接问题

**Q1：接口返回的 null 字段导致 AI 回答异常**

部分字段（`remark`、`materialBottlenecks`、`capacityBottlenecks`）必须按规范返回空字符串或空数组，不能返回 `null`。

```json
// ❌ 错误
"remark": null,
"materialBottlenecks": null

// ✅ 正确
"remark": "",
"materialBottlenecks": []
```

**Q2：gapDays 正负值定义**

```
gapDays = scheduledDate - dueDate（天数差）
正数 = 模拟交期比期望交期晚 = 延迟
负数 = 模拟交期比期望交期早 = 提前
```

**Q3：delayedTop10 排序要求**

`delayedTop10` 数组必须按 `delayDays` 降序排列（延迟天数最多的排在最前）。

**Q4：前端 CORS 问题**

如果前端直接调用 AI 助手服务出现跨域，建议通过客户方后端做代理转发，避免将 AI 服务地址暴露在前端。

## 附录：错误码说明

| HTTP 状态码 | code                 | 说明               | 处理建议       |
| -------- | -------------------- | ---------------- | ---------- |
